# 果園收支管理系統：開發計畫與技術手冊 (Updated)

## 1. 系統願景
本系統旨在為果園開發一套具備現代化 UI、高效數據統計與 Google Sheets 同步功能的移動端優先 Web App。透過視覺化的統計卡片與靈活的篩選機制，協助管理員快速掌握收支細節。

---

## 2. 核心功能頁面與近期更新

### 📊 複合式收入儀表板 (Composite Income Card) [NEW]
- **整合統計**：將「市場收入」與「客戶訂單」整合至單一統計卡片。
- **堆疊條形圖 (Stacked Bar)**：視覺化呈現市場與客戶收入的比例關係。
- **未出貨警報**：自動統計 pending 狀態的訂單，並依品種顯示警戒圖示 (🍅/🍊/🍑)。

### ➕ 全域懸浮按鈕 (Global FAB) [NEW]
- **快速入口**：位於螢幕右下角，點擊展開新增選項。
- **三向直達**：直接開啟「新增市場收入」、「新增客戶訂單」或「新增支出」表單。
- **動態交互**：具備旋轉動畫切換與毛玻璃選單效果。

### 🍎 市場收入 (Market Income)
- **統計卡片**：按品種 (甜柿、水蜜桃、橘子) 顯示卡片，卡片顏色隨品種變換。
- **紀錄複製**：新增「複製一筆」功能，快速帶入現有紀錄規格。

### 🛒 客戶訂單 (Orders)
- **品種統計**：首創訂單統計區，一目了然各品種的預訂金額與未交貨筆數。
- **自動算價**：連動「販售金額設定」自動帶入含運售價。

### 💸 支出管理 (Expense)
- **雙排統計卡片**：
  - 第一排：主類別統計 (工人薪資、資材、伙食)。
  - 第二排：個別工人薪資統計，快速看清每位工人的待付金額。

---

## 3. 技術規格與數據架構

### 🏗️ 系統架構
- **前端**：Vanilla JS + CSS3 (Glassmorphism 風格) + HTML5。
- **後端**：Google Sheets API (v4)。
- **認證**：Google OAuth 2.0 (Identity Service)。

### 📁 資料儲存結構 (Google Sheets)
- **USERS**：儲存 Email 與角色 (Admin/User)。
- **SETTINGS**：系統預設與自定義類別、工人清單、販售定價。
- **INCOME**：市場批發紀錄。
- **EXPENSE**：包含薪資、資材與伙食的詳細支出。
- **ORDERS**：客戶訂單明細。

---

## 4. UI/UX 設計規範
- **Stat Cards**：所有分頁統一使用卡片式總覽，具備 `border-top` 色條區分與懸停動畫。
- **Quick Shortcuts**：在表單日期欄位旁加入「今天」快捷按鈕。
- **Palette**：
  - 🍎 甜柿/資材：綠色系 (`#22c55e`)
  - 🍑 水蜜桃/工人：藍色系 (`#3b82f6`)
  - 🍊 橘子/其他：紫色系 (`#a855f7`)
  - ⚠️ 支出警示：紅色系 (`#ef4444`)

---

## 5. 驗證與測試項目
1. **數據連通性**：確保 Google Sheets 數據正確加載至複合式卡片。
2. **FAB 交互**：點擊 FAB 需流暢彈出選單，背景需有毛玻璃效果。
3. **複製功能**：點擊「複製一筆」後，表單需準確帶入所有非日期的欄位。
4. **響應式配置**：確保在手機版下堆疊條形圖與 FAB 位置正確，不遮擋重要資訊。
