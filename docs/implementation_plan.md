# 假日果園時薪計帳系統實作計畫

本計畫旨在建立一個以手機操作為主、資料儲存在 Google Sheets 的「工人時薪計帳系統」。為了解決目前的程式碼問題，我們將重新設計架構，使其更符合實務需求。

## 系統角色與權限架構

透過 Google OAuth 2.0 登入。我們會在 JavaScript 內獲取使用者的登入 Email。
一旦使用者的 Email 符合在系統中預先綁定的「管理員名單」，系統便會自動解鎖管理員專屬的設定介面。
採用長時間保留 Token 的機制 (localStorage)，解決頻繁需要重新登入的痛點。

## 功能規格設計

### 👨‍🌾 一般使用者 (User)
* **快速新增紀錄：** 直覺的手機介面，包含日期（預設今日）、工人姓名、工作項目（類別/次類別）、時數/天數、時薪與備註。
* **增刪改查 (CRUD)：** 能夠新增自己建立的工時紀錄。
* **備註功能：** 可以對每筆紀錄加入詳細說明（例如：提早下班、借支等）。
* **歷史追蹤：** 清楚顯示近期新增的工時明細與小計。

### 👑 管理者 (Admin)
* **包含使用者所有功能。**
* **專屬身分驗證：** 透過指定的 Google 帳號授權直接解鎖。
* **類別與選項管理：** 能夠設定下拉選單的選項內容（主類別、次類別如："疏果"、"剪枝"、"套袋"）。
* **客製化內容設定：** 設定系統常用的參數（常備工人名單）。
* **LINE 複製小幫手：** 一鍵統整並複製指定日期或工人的薪資明細，生成容易閱讀的文字格式，方便直接貼上至 LINE。

## Google 試算表結構設計

在目標試算表中建立兩個獨立的**工作表 (Sheets)** 標籤：

### 1. 工作表名稱：`Records` (儲存工時紀錄)
請在試算表中建立以下標題作為第一列 (A1~I1)：
1.  **ID** (唯一碼)
2.  **Date** (工作日期)
3.  **WorkerName** (工人姓名)
4.  **MainCategory** (主類別，例如：工人、另外支援)
5.  **SubCategory** (次類別，例如：疏果、剪枝、套袋)
6.  **Hours** (工作時數)
7.  **HourlyRate** (時薪/日薪)
8.  **Notes** (備註)
9.  **Timestamp** (最後更新時間)

### 2. 工作表名稱：`Settings` (儲存參數與下拉選單設定)
這部分讓系統動態抓取管理者設定的參數，標題為第一列 (A1~B1)：
1.  **SettingType** (設定類型，如 "Worker", "MainCategory", "SubCategory")
2.  **SettingValue** (設定值，例如工人名字或類別名稱)

## 技術與架構實作方案

1.  **前端介面：** 純 HTML5 + 原生 JavaScript (Vanilla JS)。以 CSS3 的 RWD 響應式設計為核心。
2.  **Google 登入與身份機制 (改進)：**
    *   使用 `google.accounts.oauth2.initTokenClient` 進行 OAuth 授權，取得能讀寫 Sheets 並獲取 UserInfo 的權限範圍 (`scopes: profile email, spreadsheets`)。
    *   **持久化登入 (Persistent Session)：** 將拿到的 Access Token 與計算後的過期時間存放在瀏覽器的 `localStorage` 中。載入時若 Token 尚未過期，則直接取用以進入系統，避免頻繁彈窗。
    *   **後台自動判斷管理者：** 取用 Token 後立刻呼叫 `oauth2/v3/userinfo` 確認使用者的 Email，若存在於白名單則開放 Admin 面板。
